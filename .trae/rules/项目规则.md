# 项目规则

> 任何对话中都应该遵守的硬约束。

## 一、关键安全

1. **禁止硬编码 API Key**：高德 Key、智谱 Key 一律通过环境变量注入，不允许源码中出现真实 Key 值
2. **部署必须排除 `api/data`**：打包时排除 `api/data` 目录，防止本地空数据库覆盖服务器用户数据
3. **`.env` 不入库**：`.gitignore` 已排除所有 `.env`，只保留 `.env.example` 模板

## 二、地图状态机

地图全局单例，通过 CSS 类切换四种状态，**每个页面离开时必须还原**：

| 状态 | 页面 |
|------|------|
| `map-sleep` | 首页、其他无地图页面 |
| `map-active` | 生成方案页 |
| `map-pitfalls` | 避坑指南页 |
| `map-result` | 结果页 |

**任何页面的 useEffect cleanup 必须执行**：
- `mapEl.classList.remove(...当前状态, ..., "map-sleep")`
- `clearPitfallMarkers()` 和/或 `clearPolygons()`
- `sleepMap()`

## 三、环境变量

服务器没有 `.env` 文件，所有环境变量通过 Supervisor 的 `environment=` 注入（`/etc/supervisor/conf.d/csg-api.conf`）：

| 变量 | 说明 |
|------|------|
| `AMAP_WEB_KEY` | 高德 Web 服务 Key |
| `ZHIPUAI_API_KEY` | 智谱 AI Key |

本地开发用 `api/.env`（不入库）。
